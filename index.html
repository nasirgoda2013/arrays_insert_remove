Bug #1
var gems = ["red", "blue", "green"] print(gems[3]) // 🐛 BUG!
✅ Index 3 does not exist! The array has 3 items, so the last index is 2.
Fix: gems[2] → returns "green"
Bug #2
var moves = ["jump", "run"] moves.count = 3 // 🐛 BUG!
✅ .count is read-only! You cannot set it. It only tells you the number of items.
Use moves.append("walk") to add items instead.
Bug #3
var list = ["a", "b", "c"] list.Append("d") // 🐛 BUG!
✅ Wrong capital letter! Swift is case-sensitive.
Fix: list.append("d") — lowercase "a"
📖 Focused Instruction
Removing Items: remove(at:)

Syntax

arrayName.remove(at: index)

This removes the item at the given index number. All items after it shift left by one.

⚠️ Key Point!

📖 Focused Instruction
Inserting Items: insert(_:at:)

Syntax

arrayName.insert("item", at: index)

This puts a new item at the given index. All items at or after that position shift right.

💡 Remember

insert() pushes other items to the right. append() only adds to the END.

All 5 Array Features
🔍

array[index]

🔢

.count

Count how many items are in the array

gems.count → 3
➕

.append(item)

Add a new item to the END

gems.append("gold")
❌

.remove(at:)

Remove item at a specific index

gems.remove(at: 1)
📌

.insert(_:at:)

Add item at a specific position

gems.insert("new", at: 0)
⚠️ Watch Out
Common Beginner Mistakes
❌ gems[4] // only 4 items

Index out of range! For 4 items, max index = 3 (start from 0).

✅ gems[3]
❌ gems.count = 5

.count is read-only. You can't change it.

✅ gems.append("x")
❌ gems.remove(4)

Must write at: before the index number.

✅ gems.remove(at: 4)
❌ gems.Insert("x", at: 0)

Capital "I"! Swift is case-sensitive.

✅ gems.insert("x", at: 0)

Question 1 of 6

After insert("gold", at: 0), what is portals[0]?

insert(_:at: 0) puts the new item at the very beginning of the array.
🎉 Correct! "gold" is inserted at index 0, pushing everything else right.
❌ Not quite. insert("gold", at: 0) places "gold" at the very beginning. The answer is A) "gold".
Question 2 of 6

After remove(at: 2), what is portals.count?

After insert, we have 4 items. Then remove takes one away. 4 − 1 = ?
🎉 Correct! We started with 3, inserted one (4), then removed one (3).
❌ Not quite. Start: 3 items → insert makes 4 → remove makes 3. The answer is B) 3.
Question 3 of 6

What is portals[portals.count - 1] after all changes?

count - 1 gives you the LAST index. After all changes, the array is ["gold", "red", "green"].
🎉 Correct! The final array is ["gold", "red", "green"]. The last item is "green".
❌ Not quite. After insert & remove: ["gold", "red", "green"]. count-1 = 2, so portals[2] = "green". Answer: D.
Question 4 of 6

Which is the correct syntax to add "gold" to the end?

Swift is case-sensitive! And append() uses only the item as a parameter — no at:.
🎉 Correct! append() — lowercase, one argument, adds to the end.
❌ Not quite. The correct syntax is gems.append("gold") — lowercase "a", no "at:" parameter. Answer: D.
Question 5 of 6

var g = ["a","b","c"] — After g.remove(at: 1), what is g.count?

We start with 3 items and remove one. How many are left?
🎉 Correct! 3 items minus 1 removed = 2 items left.
❌ Not quite. We had ["a","b","c"] (3 items), removed one → 2 items left. Answer: A.
Question 6 of 6

insert("x", at: 2) puts "x" at which position?

The at: parameter tells you exactly where the new item goes!
🎉 Correct! insert(_:at: 2) places the item exactly at index 2.
❌ Not quite. at: 2 means position 2. Answer: B.
